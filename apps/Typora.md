Typora 是我比较喜欢的 Markdown 编辑器，优点是边写边渲染，适合写文档、博客草稿和笔记，不需要在源码视图和预览视图之间来回切换。

## 适合什么场景

- 写博客草稿
- 整理课程笔记
- 维护项目文档
- 导出 PDF 或其他格式的文稿

## 我看重的点

- 所见即所得，写作过程更顺手
- 对表格、代码块、公式的支持比较完整
- 本地文件管理简单，适合长期积累笔记

## 安装

- 官方网站：[typora.io](https://typora.io/)
- 安装后建议先确认主题、字体、代码块样式和图片保存策略

## 使用建议

1. 固定一套标题层级和命名方式，后期整理会轻松很多。
2. 图片和附件尽量统一放到单独目录，避免文档搬家时路径混乱。
3. 如果后续要迁移到知识库或 Git 仓库，优先使用标准 Markdown 语法。
